# 医院预约挂号管理系统 - 安装和使用说明

## 一、项目简介

这是一个基于 Spring Cloud 微服务架构和 Vue3 的医院预约挂号管理系统，用于解决传统医疗挂号排队时间长、号源分配不均等问题，提供高效的挂号、排班、医生与患者管理服务。

### 主要功能

- **科室管理**：管理医院科室信息
- **医生管理**：管理医生信息和排班
- **患者管理**：管理患者信息
- **预约挂号**：患者在线预约挂号
- **预约审核**：管理员审核取消申请
- **就诊记录**：记录患者就诊信息

## 三、数据库配置

### 3.1 MySQL 数据库初始化

1. **创建数据库**
```sql
CREATE DATABASE IF NOT EXISTS ry-cloud DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

2. **导入 SQL 脚本**
- 执行项目根目录下的 `sql/` 目录中的 SQL 文件
- 执行 `init_admin_data.sql` 初始化管理员数据

### 3.2 Nacos 配置中心

1. **启动 Nacos**
```bash
cd nacos/bin
startup.cmd
```

2. **访问 Nacos 控制台**
- 地址：http://localhost:8848/nacos
- 默认账号：nacos / nacos

3. **导入配置**
- 在 Nacos 控制台中导入配置文件
- 配置文件包括：数据库连接、Redis 连接、服务端口等

### 4.2 后端打包步骤

#### 使用 IDEA Maven 面板

1. **打开 Maven 面板**
   - 点击 IDEA 右侧边栏的 `Maven` 标签

2. **清理之前的构建**
   - 在 `Maven` 面板中展开 `ruoyi` 项目
   - 展开 `Lifecycle`
   - 右键点击 `clean`，选择 `Run Maven Build`
   - 等待清理完成（显示 `BUILD SUCCESS`）

3. **打包项目**
   - 在 `Maven` 面板中展开 `ruoyi` 项目
   - 展开 `Lifecycle`
   - 右键点击 `package`，选择 `Run Maven Build`
   - 等待打包完成（显示 `BUILD SUCCESS`）

#### 使用命令行

```bash
# 进入后端项目目录
cd RuoYi-Cloud

# 执行打包命令
mvn clean package -DskipTests
```

#### 使用 IDEA Terminal

3. **展开 `Scripts`**
4. **双击 `build:prod`**
5. **等待打包完成**

打包成功后，静态文件位于 `dist` 目录：

```
RuoYi-Cloud-Vue3/

    ├── index.html              # 主入口文件
    ├── favicon.ico             # 网站图标
    └── static/                 # 静态资源目录
        ├── css/                # 样式文件
        ├── js/                 # JavaScript 文件
        └── 图片文件
```

### 5.4 前端部署

#### Nginx 部署

1. **安装 Nginx**
   - 下载 Nginx for Windows 并解压

2. **复制文件到 Nginx 目录**
```bash
# Windows
xcopy RuoYi-Cloud-Vue3\dist\* C:\nginx\html\ /E /I
```

3. **配置 Nginx**

编辑 `nginx.conf` 文件：

```nginx
server {
    listen 80;
    server_name localhost;

    location / {
        root html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 代理后端 API
    location /prod-api/ {
        proxy_pass http://localhost:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

4. **启动 Nginx**
```bash
cd nginx
start nginx
```

1. **登录系统**：访问系统首页，使用管理员账号登录

4. **审核申请**：审核患者的取消申请，处理异常情况

## 八、常见问题

### 8.1 后端问题

#### 问题 1：Maven 打包失败

**现象**：依赖下载失败或编译错误

**解决方法**：
1. 检查 Maven 配置
2. 清理本地仓库：`mvn clean`
3. 强制更新依赖：`mvn clean package -U`
4. 检查 Java 版本（需要 Java 17）

#### 问题 2：服务启动失败

**现象**：服务启动失败，报错连接不上 Nacos/Redis/MySQL

**解决方法**：
1. 检查 Nacos、Redis、MySQL 是否已启动
2. 检查端口是否被占用
3. 查看日志文件定位错误
4. 检查配置文件中的数据库、Redis 连接信息

### 8.2 前端问题

#### 问题 1：npm install 失败

**现象**：依赖安装失败

**解决方法**：
1. 删除 `node_modules` 和 `package-lock.json`
2. 重新安装依赖：`npm install`
3. 检查 Node.js 版本（建议 16+）
4. 使用淘宝镜像：`npm config set registry https://registry.npmmirror.com`

#### 问题 2：打包失败

**现象**：Vite 打包失败

**解决方法**：
1. 检查环境变量配置
2. 检查 API 地址配置
3. 清理缓存：`rm -rf node_modules/.vite`
4. 重新打包：`npm run build:prod`

#### 问题 3：部署后页面空白

**现象**：Nginx 部署后访问页面空白

**解决方法**：
1. 检查 Nginx 配置中的 `root` 路径
2. 检查 `try_files` 配置
3. 检查浏览器控制台是否有错误
4. 检查 API 代理配置是否正确

### 8.3 数据库问题

#### 问题 1：数据库连接失败

**现象**：服务启动时报数据库连接失败

**解决方法**：
1. 检查 MySQL 是否启动
2. 检查数据库用户名和密码
3. 检查数据库名称是否正确
4. 检查防火墙是否阻止连接

### 8.4 Redis 问题

#### 问题 1：Redis 连接失败

**现象**：服务启动时报 Redis 连接失败

**解决方法**：
1. 检查 Redis 是否启动
2. 检查 Redis 端口（默认 6379）
3. 检查 Redis 密码配置
4. 检查防火墙设置
